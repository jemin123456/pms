## Recommended MVP for a 48-hour Odoo Hackathon

Trying to build all of the above in a short hackathon isn't realistic. Focus on these high-impact features:

1. User roles and authentication
2. Project management
3. Task management with Kanban
4. Calendar and Gantt views
5. Dashboard with charts
6. Time tracking
7. Notifications
8. Reports (PDF/Excel)
9. Chatter and activity tracking
10. One AI feature (such as automatic task prioritization or AI project summaries)

This combination demonstrates strong knowledge of Odoo's framework while remaining achievable within a hackathon timeline.

Task Module.

--------------------------------------

IMPORTANT

Do NOT generate the entire project in one response.

Generate ONE module at a time.

Wait for approval before moving to the next module.

Every generated code must be production ready.

Every function must include proper validation, authorization, error handling, logging, comments, and modular separation.

Whenever writing a controller, NEVER access MongoDB directly.
Controllers call services.
Services call repositories.

Policies must be called before business logic.

Repositories contain database queries only.

No business logic inside routes.

Follow this architecture consistently throughout the project.
